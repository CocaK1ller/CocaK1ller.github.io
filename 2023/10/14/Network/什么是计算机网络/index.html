什么是计算机网络

CocaKiller Lv2
  2023-10-14 16:33 2023-10-14 16:33 Created   2023-10-15 00:09:08 2023-10-15 00:09:08 Updated  
  • 计算机网络 
 

计算机网络 = 通信技术 + 计算机技术

一种特殊的通信网络：信源和信宿通常是计算机，信息是数字化的

  • 互联的、自治的计算机集合

    • 自治：无主从关系
    • 互联：互联互通
      • 通信链路：无线信号/电、光纤、铜缆、卫星

  • 距离远、数量大如何保证互联

    • 直接链路连接不可行

  • 通过交换网络互联主机

  • 交换节点

    • 计算机网络中指路由器或交换机

什么是Internet——组成细节

  • 互联网是最大的计算机网络
  • 数以百万计的互联的计算设备集合
    • 主机(hosts) = 端系统(end system) | PC，服务器，无线笔记本
    • 运行各种网络服务

  • 分组交换：转发分组（数据包）
    - 路由器（routers）和交换机（switches）

服务角度

  • 为网络应用提供通信服务的通信基础设施
  • 为网络应用提供应用编程接口（API）
    • 支持app连接internet，发送/接收数据

仅有硬件连接，Internet能否顺畅运行？能保证应用数据有序交付吗？

  • NO！还需要**协议**

协议

学习网络的重要内容之一
网络创新的表现形式之一

协议是计算机网络有序运行的重要保证

  • 硬件是计算机网络的基础
  • 计算机网络中的数据交换必须遵守事先约定好的规则
  • 如同交通规则

发送特定消息

采取特定动作

网络协议（Network Protocol）简称为协议，是为进行网络中的数据交换而建立的规则、标准或约定

协议规定了通信实体之间所交换的信息的格式、意义、顺序以及针对收到信息或发生的事件所采取的“动作“（action）

协议规范了网络中所有信息发送和接受过程

三要素

  • 语法（Syntax）/信件
    • 数据与控制信息的结构或格式
    • （对于底层信息）信号电平
  • 语义（Semantics）/信封
    • 需要发出何种控制信息
    • 完成何种动作以及做出何种响应
    • 差错控制
  • 时序（Timing）
    • 时间顺序
    • 速度匹配

计算机网络结构

  • 网络边缘

    • 主机
    • 网络应用

  • 接入网络，物理介质

    • 有线/无线

  • 网络核心（核心网络）

    • 互联的路由器（或分组转发设备）
    • 网络之网络

  • 主机（端网络）

  • 客户/服务器应用模型：

    • 谁主动发起通信，谁就是客户
    • 客户和客户之间不会通信

  • 对等（peer-peer, P2P）应用模型

    • 五（或不仅依赖）专用服务器
    • 通信在对等实体之间进行
    • QQ，Skype

  • 接入网络

    • 可能是一个复杂的网络，或简单的通信链路

    • 解决的问题：如何将网络边缘接入核心网（边缘路由器）

      • 接入网络（家用、机构、移动……)

    • 用户关心的是：

      • 带宽：数据传输速率（而不是信号最高最低之差）
      • 共享/独占（带宽是否共同使用）

    • DSL：数字用户线路

      • 多路信号共同使用一个物理介质
      • 独占至中心局的接入

      考虑到用户上网通常需要获取网上的信息而非发布信息，所以设计上使得下载速度比上传速度快。

      • ADSL：非对称数字用户新路

    • 电缆网络

      • 频分多路复用：在不同频带（载波）上传输不同频道
      • 共享家庭至电缆头端的接入网络

  • 机构接入网络

    • 以太网：计算机局域网技术

  • 无线接入网络

    • 通过共享的无线接入网络连接端系统与路由器
    • 无线局域网（LANs）
    • Wi-Fi
    • 广域无线接入
      • 蜂窝网
      • 3G、4G

网络核心

  • 互联的路由器网络
  • 关键功能：路由 + 转发

    • 根据地址信息，把数据通过接口送到下一个节点

    • 路由器：本地转发表/路由表

      • 根据路由协议，运行路由算法得到的

    • 路由（routing）： 确定分组从源到目的传输路径

    • 转发（forwarding）：将分组从路由器的输入端口交换至正确的输出端口

  • 解决的基本问题
    • 如何实现数据从源主机通过网络核心送达目的主机
      • 数据交换

Internet结构：网络之网络

端系统（主机）通过接入ISP（access ISPs，互联网接入服务提供商）连接到Internet

  • 接入ISP必须进一步互联
    • 这样任意两个主机才可以互相发送分组
  • 构成复杂的网络互联的网络
  • 当前结构？
    • 无人能给出精确描述

可选方案

  • 每个接入ISP直接彼此互联

    • 直接互联不适用于大规模网络：O(n^2)连接问题

  • 每个接入ISP连接到全球ISP

    • 必定有竞争者
    • 这些ISP必须互联
      • 对等链路（peering link）
      • 互联网交换节点（IXP， exchange point）
        • 本身就是一个高速的网络
      • 区域网络，区域ISP
      • 内容提供商网络
        • 如google，微软
        • 可能运行自己的网络，并就近为端用户提供服务、内容
        • 通常绕过一级ISPs和区域ISPs进行构建
