Hexo 字体修改

Arno Yang Lv1
  2025-01-23 2025-01-23 Created   2025-01-24 16:05:53 2025-01-24 16:05:53 Updated  
  • 前端 
 

这篇文章主要记录在配置hexo的字体格式时遇到的坑。

目的

看到 Redifine 中提供了很简单的配置，用来修改网页中的中/英文/代码字体，所以决定尝试一下。

步骤

首先需要了解一下字体(fonts)属性下的配置

  • url: 文档中的链接索引到了这样一个CSS，用来自定义一个字体。
  • family: 对应CSS中的font-family，作用是用一个列表存储family相同的字体配置，提供一些相对接近的备选字体。看上去只要将CSS文件中的font-family与这里的family保持相同就可以了

按照上面谷歌api提供的字体配置，我们需要提供一个类似的CSS文件，其中src字段表示最后的字体文件的存放地址。

存放地址选用的是themes\redefine\source\fonts文件夹，这个文件夹下存放了三个主题默认字体。直接仿照他们的CSS文件配置：

之前看到google提供的均为woff2格式的字体文件。WOFF与WOFF2是两种压缩字体格式，因此三种字体的兼容性是 TTF > WOFF > WOFF2，体积则相反。浏览器对字体的兼容性可以查看这个网址。

踩到的坑

一开始将font-family与font的值均设置为”宋体w7”，并没有成功在页面显示设置的字体。通过 Chrome + F12 发现是CSS文件并没有成功读取到指定url的文件，这里是因为yml格式中引号虽然是可选的，但对于特殊字符和空格还是需要使用引号避免解析错误。之后测试了一些font-family与font的组合，结果如下：

font-family font 结果
宋体w7 宋体w7 true
宋体w7 宋体 false
宋体 宋体w7 true
宋体w7 宋体w7w false
宋体 宋体宋体 true
w7 w7 true
w7 w7w false

开发者平台的警告：'content-type' header charset value should be 'utf-8'.

这似乎是一个索引问题，font-family可以索引到同名的font。除此以外，当font-family为中文时，只会检索font是否出现同样的中文前缀。

GPT的解释：

  • 字体名称解析：浏览器在解析 font-family 和 font 时，可能有不同的解析机制。如果 font-family 中的名称与系统或网页加载的字体库中的字体名称完全匹配，字体会正确应用。
  • 中文前缀匹配：对于中文字体名称，有可能浏览器在某种情况下采用前缀匹配，即 font-family 值中的中文部分匹配字体文件名称中的中文前缀时，字体也会被正确应用。

结论

最后没有去深入研究究竟是哪里的解析出现的问题，但是再次回忆起被中英文混用支配的恐惧……以后尽量还是英文编码吧。
